# HavenIntel Tomorrow Plan

Date: June 13, 2026

## Goal

Put HavenIntel on the real domain as a **public soft launch**, while keeping paid member operations closed until the backend layer is safer.

## Launch mode

Tomorrow is **not** a full paid launch.

Tomorrow **is**:

- real domain
- real public pages
- real live-board viewing
- real archive viewing
- real insights publishing
- real mobile and device testing

Tomorrow is **not yet**:

- real public checkout
- real package fulfillment
- real paid member delivery
- real admin-led paid operations

## What stays public

- homepage
- pricing section
- live board
- past matches
- insights
- article pages
- privacy and policy pages

## What stays visually closed

- package unlock flow
- public paid CTA path
- real commercial member onboarding

## Tomorrow sequence

## 1. Connect the domain

- connect `haven-intel.com`
- confirm homepage loads correctly
- confirm asset paths work on the real host
- confirm article pages load on the real host

## 2. Verify public navigation

- homepage
- pricing anchor
- live board
- past matches
- insights
- policy links

Check:

- mobile nav wrapping
- sticky header spacing
- no broken links

## 3. Verify public page quality

Check on desktop and mobile:

- homepage hierarchy
- pricing cards
- live board status legend
- tentative / confirmed / released labels
- archive ordering
- insight cards
- article layout
- footer layout

## 4. Confirm paid flow stays hidden

- no public nav push into checkout
- homepage package buttons show `Opening Soon`
- member path is not presented as active commerce
- no accidental live payment wording

## 5. Publish one strong insight

- keep Canada vs Bosnia pre-match article live
- make sure the article reads editorial, not like a betting slip
- confirm published date appears

## 6. Real-world testing pass

Test:

- Safari
- Chrome
- mobile browser
- different screen widths

Watch for:

- layout breaks
- anchor scroll issues
- text overflow
- card stacking issues
- sticky nav overlap

## 7. Gather first issue list

Create a short list under 3 buckets:

- visual issues
- content/copy issues
- workflow/logic issues

## 8. Decide next branch

After soft launch feedback, decide:

### Branch A

Continue polishing public UX only.

### Branch B

Start production layer work:

- backend auth
- email delivery
- Stripe
- admin protection
- persistent database state

## What would make tomorrow successful

Success tomorrow does **not** mean selling packages.

Success tomorrow means:

- the site looks credible on the real domain
- the public experience feels stable
- the board and archive make sense
- the content reads well
- we learn what breaks in the real environment

## What would make tomorrow a mistake

- accepting money before access is real
- opening checkout before delivery is real
- presenting preview member flow as if it is production-safe

## Tomorrow close-out

At the end of tomorrow, we should have:

- a real-domain public site
- a bug list from live usage
- a decision on whether to start production backend and Stripe immediately
